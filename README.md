# Git 前端界面

一个现代化的、基于Web的Git用户界面，提供直观的方式来管理Git仓库。

## ✨ 功能特性

- 📝 **提交历史** - 查看详细的提交记录，包括作者、日期和提交信息
- 🌿 **分支管理** - 创建、切换和删除分支
- 📊 **工作区状态** - 实时查看文件修改状态（已修改、已暂存、未跟踪等）
- 🔍 **差异对比** - 查看工作区、暂存区和HEAD之间的差异
- 🎨 **现代化UI** - 响应式设计，优雅的用户界面
- ⚡ **实时更新** - 快速刷新和加载

## 📋 系统要求

- Node.js 14.x 或更高版本
- Git 2.x 或更高版本
- 现代浏览器（Chrome、Firefox、Safari、Edge）

## 🚀 快速开始

### 安装依赖

```bash
npm install
```

### 启动服务器

```bash
npm start
```

或使用开发模式（自动重启）：

```bash
npm run dev
```

### 访问界面

打开浏览器访问：`http://localhost:3000`

## 🔧 配置

### 自定义Git仓库路径

默认情况下，服务器会使用当前目录作为Git仓库。你可以通过环境变量指定其他路径：

```bash
GIT_REPO_PATH=/path/to/your/repo npm start
```

### 自定义端口

```bash
PORT=8080 npm start
```

## 📖 使用说明

### 提交历史

1. 点击左侧导航栏的"提交历史"
2. 输入要显示的提交数量（默认20条）
3. 点击"加载提交"按钮查看历史记录

### 分支管理

1. 点击左侧导航栏的"分支管理"
2. 查看所有分支，当前分支会高亮显示
3. 输入新分支名称并点击"创建分支"来创建新分支
4. 点击"切换"按钮切换到其他分支
5. 点击"删除"按钮删除不需要的分支

### 工作区状态

1. 点击左侧导航栏的"工作区状态"
2. 系统会自动显示所有文件的状态
3. 文件会按照状态分组显示：
   - 已暂存（STAGED）
   - 已修改（MODIFIED）
   - 未跟踪（UNTRACKED）
   - 已删除（DELETED）

### 差异对比

1. 点击左侧导航栏的"差异对比"
2. 选择对比类型：
   - 工作区 vs 暂存区
   - 暂存区 vs HEAD
   - 工作区 vs HEAD
3. （可选）输入特定文件路径
4. 点击"查看差异"按钮

## 🔌 API端点

### 分支操作

- `GET /api/branch/current` - 获取当前分支
- `GET /api/branches` - 获取所有分支
- `POST /api/branch/create` - 创建新分支
- `POST /api/branch/checkout` - 切换分支
- `POST /api/branch/delete` - 删除分支

### 提交和状态

- `GET /api/commits?limit=20` - 获取提交历史
- `GET /api/status` - 获取工作区状态
- `GET /api/diff?type=working&file=path` - 获取差异

### 健康检查

- `GET /health` - 服务器健康状态

## 🎨 界面预览

- **顶部导航栏** - 显示当前分支和刷新按钮
- **左侧边栏** - 功能导航菜单

## 🐛 故障排除

### 无法连接到服务器

确保服务器正在运行：
```bash
npm start
```

### Git命令执行失败

确保你在Git仓库目录中运行服务器，或者设置正确的`GIT_REPO_PATH`环境变量。

### 端口被占用

使用不同的端口：
```bash
PORT=8080 npm start
```
